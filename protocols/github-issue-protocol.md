GitHub Issue Use Protocol

Purpose: This protocol explains how to use GitHub Issues to track group tasks, questions, bugs, discussion items, and decisions. Issues help the team keep work visible, assign responsibility, record context, and connect tasks to branches and pull requests. General rule: create an issue before starting a meaningful repository update, especially when the task affects scripts, notebooks, documentation, protocols, data cleaning, analysis workflows, or project decisions.

1. **Steps**

   1. Go to our GitHub repo
   2. Click Issues, then click New issue.
   3. Write a title for it and add a description

      1. Suggested title format: short-description, eg. 01-update-issue-protocol (GitHub automatically gives each issue a number)
   4. (Assign the issue to the person responsible and add labels if needed)
   5. Click Submit new issue
2. **Common issue types**

   1. docs: for documentation and protocol updates
   2. bug: for code or workflow problems
   3. analysis: for analysis tasks
   4. data-cleaning: for cleaning or metadata tasks
   5. question: for questions needing team input
   6. discussion: for broader decisions
